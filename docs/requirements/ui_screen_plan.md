# 画面設計メモ（MVP / iPhone優先）

## 決定事項（作品一覧）
- 作品一覧は **グリッドではなく1行リスト** を採用する。
- 各行は「タイトル中心」で、視認性とタップしやすさを優先する。
- 参考イメージの「インターネットなし」風の軽いステータス行を、一覧上部（タイトル直下）に表示できる設計にする。
- 一覧画面の右上アクションは `...` ではなく **歯車（設定）** にする。

## 作品一覧セル（1行）の最小構成
- 1行目: 作品タイトル（太字）
- 2行目: 更新日時（例: `2026/02/07 14:20`）
- 3行目: 種別ラベル（例: `小説`）または補足（任意）
- 右端: 必要なら `>`（詳細遷移）

## 一覧画面の固定要素
- 上部: 画面タイトル
- 右上: 歯車アイコン（タップでSettingsへ直接遷移）
- タイトル直下: 同期ステータス行（条件表示）
  - 確認中: ローカル保存のみで操作継続 + 小さい`ProgressView`を表示
  - 表示条件: `localOnlyBanner`（iCloud同期不可）
  - 非表示: `syncEnabled`
- 下部: 検索バー + 新規作成導線（既存UI方針に合わせる）

## 状態別表示（MVP-3連動）
- `syncEnabled`: 通常の一覧表示
- `localOnlyBanner`: 一覧は使える + 同期不可ステータス行を表示
- `requiresICloudSignIn`: 一覧には入れず、iCloudサインイン誘導画面へ遷移
- `判定中`: ホームは操作可能のまま、同期ステータス行のみ「確認中表示」にする

## デバイス展開方針
- iPhone: 1カラム遷移（一覧 -> 編集 -> 履歴）
- iPad/mac（将来）: 同じ情報設計を2〜3カラムに拡張する（情報は増やさず配置のみ変更）

## 決定事項（エディタ画面）
- 左上の戻るボタンの右側に、現在編集中の**ファイル名**を表示する。
- 下部ツールバーは左下に3ボタン、右下に1ボタンの構成にする。
- 右下ボタンは「新規作成」ではなく**ブランチ切替**にする。

## エディタUI（固定構成）
- 上部ヘッダー
  - 左: 戻るボタン
  - 戻るの右: ファイル名（長い場合は省略）
  - 右: 元に戻す / 共有 / その他メニュー（現行方針を維持）
- 左下ツールバー（左から順）
  - 1: 簡易エクスプローラー（ファイル/フォルダ一覧）
  - 2: 変更一覧（例: `sample.md +29 -15` のような表示）
  - 3: 履歴グラフ（コミットグラフの簡易表示）
- 右下フローティングボタン
  - ブランチ切替（現在ブランチ表示 + 切替アクション）

## 用語モードとの連動（表示文言）
- 非技術者モード
  - Explorer: `ファイル`
  - Changes: `変更`
  - Graph: `履歴`
  - Branch: `作業ライン`（暫定）
- 開発者モード
  - Explorer: `Explorer`
  - Changes: `Changes`
  - Graph: `Graph`
  - Branch: `Branch`

## メモ
- まずは「迷わず書ける一覧」を優先し、カード化・装飾は後回しにする。
- 用語モード（非技術者/開発者）は一覧の文言にも適用可能な設計にする。
